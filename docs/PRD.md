# 產品需求文件 (PRD)：任務管理系統

## 1. 專案概述
### 1.1 背景與動機
現代人生活與工作節奏緊湊，經常有許多待辦事項需要處理。然而，散落各處的紙條或零碎的記憶往往導致任務遺漏，影響效率。因此，我們需要一個簡單、直覺的桌面任務管理系統，來集中管理所有的待辦任務，幫助使用者有效掌握生活與工作的節奏。

### 1.2 目標用戶
需要管理日常工作的個人用戶。

### 1.3 核心價值主張
提供一個輕量、易用的平台，讓使用者能快速記錄、追蹤並管理每日待辦事項，提升個人生產力與專注力。

## 2. 功能需求
系統提供以下主要功能（附使用者故事）：

### 2.1 新增任務
* **使用者故事**：作為個人用戶，我希望能夠快速新增待辦任務，以便我可以把要做的事情記錄下來以免忘記。

### 2.2 標記任務完成
* **使用者故事**：作為個人用戶，我希望點擊按鈕就能將任務的狀態標記為完成，以便追蹤我的工作進度並獲得成就感。

### 2.3 刪除任務
* **使用者故事**：作為個人用戶，我希望可以刪除不再需要或建立錯誤的任務，以便保持任務清單的乾淨與專注。

### 2.4 顯示所有任務清單
* **使用者故事**：作為個人用戶，我希望可以在首頁清楚看見所有的任務清單，以便一目了然了解我總共有哪些事情需要處理。

### 2.5 依完成狀態篩選
* **使用者故事**：作為個人用戶，我希望可以依據「全部」、「已完成」或「未完成」層級來篩選顯示的任務，以便我能更快速地專注於當下需要處理的事情。

## 3. 非功能需求
### 3.1 技術限制
* 框架與語言：使用 Python 語言搭配 Flask Web 框架開發。
* 前端模板：使用 Jinja2 模板引擎進行頁面渲染，結合基本 HTML/CSS。
* 資料庫：使用 SQLite 作為關聯式資料庫，此資料庫免安裝且方便輕量佈署與開發測試。

### 3.2 效能與安全考量
* **效能考量**：由於是供個人用戶使用的輕量系統，頁面載入時間需力求低於 1 秒。資料庫查詢應快速且操作直覺。
* **安全考量**：透過前端或後端限制防止空白或是異常惡意內容寫入系統（例如基礎的 XSS/SQL Injection 防護策略）。

## 4. MVP 範圍
本階段系統將遵循最小可行性產品（MVP）原則進行開發：

* **Must Have（必備功能）**
  * 顯示所有任務清單
  * 新增任務
  * 標記任務完成狀態

* **Should Have（應有功能）**
  * 取消完成任務的功能
  * 刪除任務
  * 依完成狀態前端篩選（全部、已完成、未完成）

* **Nice to Have（加分功能）**
  * 設定任務截止期限 (Due Date)
  * 任務標籤或優先級設定
  * 使用者註冊與登入管理 (未來擴充多使用者)
